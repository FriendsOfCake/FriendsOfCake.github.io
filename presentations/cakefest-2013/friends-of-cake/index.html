We want to create a "go to" place for high quality and well maintained CakePHP plugins.

We aim to unite people around some central, common plugins. Rather than having several forks of a plugin, we want to merge them into one awesome plugin that everyone can contribute to.
Imagine if CakeDC/users was build on CRUD, you could modify most of it without touching the base plugin, using events.

Requirements

(partial)

  • Compatible with CakePHP 2.3 and higher
  • Conform to the CakePHP code guidelines
  • At least 75% code coverage
  • Provide composer support
  • Well documented code (docblocks, @annotations etc)
  • Well documented user guide with examples
  • Comply with semantic versioning
  • High quality code (defensive coding, check return, ...)
  • Use MIT license
  • Have a project lead for each project
